# CareFusion AI Deployment Guide

This document provides instructions for accessing CareFusion AI from mobile devices and deploying it for public access.

## 1. Quick Mobile Access (Local Network)
To test the application on your mobile phone while on the same Wi-Fi network:

1.  **Find your Local IP**: Open Terminal and run `ipconfig`. Look for `IPv4 Address` (e.g., `192.168.1.5`).
2.  **Update Frontend Config**: In `carefusion_v2/frontend/src/pages/DoctorDashboard.tsx`, ensure `API_BASE` points to your local IP instead of `localhost` (or use the `.env` approach).
3.  **Run Vite with Host**:
    ```bash
    cd carefusion_v2/frontend
    npm run dev -- --host
    ```
4.  **Access from Phone**: Open your phone's browser and go to `http://192.168.1.5:5173`.

---

## 2. Public Access (Using Ngrok Tunnel)
To allow anyone on the internet to access your local machine's instance:

1.  **Install Ngrok**: Download from [ngrok.com](https://ngrok.com).
2.  **Tunnel Backend (Port 5000)**:
    ```bash
    ngrok http 5000
    ```
    *Copy the Forwarding URL (e.g., `https://random-id.ngrok-free.app`).*
3.  **Tunnel Frontend (Port 5173)**:
    ```bash
    ngrok http 5173
    ```
4.  **Update Frontend**: Set the `VITE_API_BASE` environment variable to your Backend Ngrok URL so the frontend communicates through the public tunnel.

## 3. Production Deployment (Cloud)

### Frontend (Vercel)
1.  **Preparation**: I have already added `vercel.json` in the `frontend` folder to handle routing.
2.  **Deployment**:
    *   Push your code to **GitHub**.
    *   In the **Vercel Dashboard**, click "Add New" > "Project".
    *   Import your `CareFusion-AI` repository.
    *   **CRITICAL**: Set the **Root Directory** to `carefusion_v2/frontend`.
    *   **Build Settings**:
        *   Framework Preset: `Vite`
        *   Build Command: `npm run build`
        *   Output Directory: `dist`
    *   **Environment Variables**: Add `VITE_API_BASE` and set it to your public Backend URL (e.g., your Render or Ngrok URL).
3.  **Domain**: Once deployed, go to Settings > Domains to add `www.carefusionai.com`.

### Backend (Render / Railway / Docker)
*Note: The current backend relies on local Windows paths (`C:/`). For cloud deployment, you must:*
1.  Containerize the backend using **Docker**.
2.  Bundle the Python environments (`carefusionEV`) into the image.
3.  Deploy the Docker image to a service like **Render** or **Google Cloud Run**.
4.  Ensure MongoDB is hosted on **MongoDB Atlas**.
